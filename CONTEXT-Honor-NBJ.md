## 2) 方法与输出偏好（高优先级）

### 2.1 研究方法

- 数据先行，框架随后：禁止凭记忆下财务结论。
- 关键数据要求双源交叉验证；若失败需明确披露，不可编造。
- 结论需带概率表达与反证条件（失效信号）。
- 重视二阶思考与可证伪性。

### 2.2 报告写作

- 偏好：专业投研表达，但避免模板化、机械化语言。
- 结构：前提 -> 证据 -> 推理 -> 结论（可追踪）。
- 风格：可读、叙事化、少空话。
- 关键段落需“数据+逻辑链”充分展开，不接受一句话带过。

### 2.3 输出格式

- 报告目录：`10-研究报告输出/`
- 命名格式：`[日期]-[类型]-[标的]-[框架].md`
- 交付习惯：生成 `.md` 后同步生成同路径 `.pdf`
