# 投研错误与经验教训档案

> **定位**：每次分析偏差、数据错误、框架误用后追加一条，只增不删。
> **用法**：每次开始复杂研究（L3+）前，检阅与当前标的/场景相关的条目。
> **格式**：`[日期] [错误类型] | 错误描述 → 根本原因 → 防错规则`

---

## 错误类型标签

- `[数据错误]` — 数据获取失败、数值异常、来源不可靠
- `[框架误用]` — 框架选取不当、不适用场景强行套用
- `[逻辑漏洞]` — 结论无法追溯到证据，推理断链
- `[Kill-Switch漏判]` — 红线条件已触发但未降级
- `[能力圈越界]` — 分析了不在理解范围内的标的
- `[执行问题]` — 串行等待、未先写计划、结案未验证
- `[输出质量]` — 执行摘要超标、数据未标注来源、逻辑松散

### [执行问题] 类

**[2026-02-23] [执行问题]** | L3 分析未写研究计划直接开始 → 中途发现框架选取有误，需要回炉 → **防错规则**：L3及以上，务必先写 `tasks/todo.md`，5分钟规划省30分钟返工。

**[2026-02-24] [执行问题]** | 误把 `read_file` 工具输出自带的 Markdown 围栏当作文件真实内容 → 对文件尾部做了不必要的“清理”判断 → **防错规则**：用 `read_file` 查看文件尾部时，忽略工具输出最外层的 ```markdown/``` 包裹；若怀疑文件真实存在围栏/不可见字符，用 `Get-Content -Tail` 或扩大行号范围复核。
