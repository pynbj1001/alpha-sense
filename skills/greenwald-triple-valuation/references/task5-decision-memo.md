# Task 5：买方决策备忘录 — 详细执行指引

## 目标与定位

买方决策备忘录（Buy-Side Decision Memo）是整个三重估值体系的
**唯一对外输出**——精炼、可执行、不废话。

对比卖方研报（用于营销，追求全面），买方备忘录的特质：
- **结论优先**，数据服务于结论（不是填表）
- **赔率思维**，明确三情景概率而非"中性预测"
- **可证伪**，每个判断都有可测量的反驳条件
- **说人话**，避免模型语言，聚焦"Key Variable"

---

## 强制读取前置条件

执行本任务前**必须**检查所有前序任务完成状态（见SKILL.md Task 5 验证清单）。

**关键数字必须从前序任务提取，不得重新推算**：
- AV/EPV/GV 每股数字 ← Task 3
- 护城河评级 + 最大威胁 ← Task 4
- Kill-Switch 结论 ← Task 4
- 情景概率基础 ← Task 4 三情景矩阵

### 第一节：结论优先（Executive Verdict）

**字数限制：≤3句话，禁止废话**

格式要求：
1. 句1：结论（买入/观察/回避）+ 一句理由
2. 句2：内在价值 vs 当前价格 + 安全边际
3. 句3：核心驱动变量 + 最大风险因子

**示例（好）**：
> 建议**买入**，公司宽护城河（客户粘性+规模效应）支撑EPV溢价AV约2.3倍，
> 当前价格¥45较基准EPV¥72折价37%，超越30%买入门槛。
> 核心驱动变量为毛利率持续力；最大风险为技术替代加速导致护城河期压缩。

**示例（差，禁止）**：
> 综合来看，公司具有一定竞争优势，估值相对合理，视情况可以考虑关注。（❌ 无结论，废话连篇）

**置信度标注**（必须）：
- 🟢 高置信度：估值主要依赖AV和EPV，无需依赖GV即有安全边际
- 🟡 中置信度：安全边际需要部分GV支撑；护城河有证据但持续性有不确定性
- 🔴 低置信度：安全边际完全依赖GV；估值高度敏感于增长假设

### 第二节：内在价值快照（Valuation Snapshot）

**格式**：标准表格，精确到小数点后一位，不接受区间代替数字

```
┌─────────────────────────────────────────────────────────────┐
│                    内在价值快照                               │
├──────────────────┬──────────────┬──────────────────────────┤
│ 估值维度          │ 每股价值（¥/$ ）│ 备注                     │
├──────────────────┼──────────────┼──────────────────────────┤
│ 资产价值 AV（保守）│              │ 清算/重置底线              │
│ 资产价值 AV（基准）│              │ 重置成本法                 │
│ 盈利能力价值 EPV（保守）│         │ WACC = 11%               │
│ 盈利能力价值 EPV（基准）│         │ WACC = 9%（主要参考）      │
│ 盈利能力价值 EPV（乐观）│         │ WACC = 8%                │
│ 成长价值附加 GV   │              │ 仅护城河确认时有效          │
│ 综合估值（基准）  │              │ EPV + GV（基准）           │
├──────────────────┼──────────────┼──────────────────────────┤
│ 当前市场价格      │              │ [日期]收盘价               │
│ 安全边际（EPV基准）│ [X%]        │ (EPV基准 - 价格) / 价格   │
│ EPV/AV 比值      │ [Y.Yx]       │ 护城河倍数                 │
└──────────────────┴──────────────┴──────────────────────────┘
```

### 第三节：护城河判决（Moat Verdict）

**字数：500–700字。禁止重复Task 1/4的原始描述，只提炼关键结论。**

结构：
1. **护城河评级**：[宽/中/窄/无] + 一句话理由
2. **护城河来源**：主要类型（1–2种）+ 最强量化证据（ROIC利差/转换成本/临界市占率）
3. **护城河持续性判断**：预计可维持X年，剩余护城河价值的主要支撑
4. **最大威胁**：最可能在X年内侵蚀护城河的单一最大变量，以及触发条件
5. **与Task 3 GV假设的一致性确认**：护城河期假设Y年是否与此判断吻合？

**特别要求**：
- 若护城河评级"无"或"窄"，必须明确说明GV = 0及对估值的影响
- 若Kill-Switch KS-5被触发（行业衰退），此节必须放在最突出位置警示

### 第四节：关键变量（Key Variables）

**字数：300–500字。这是买方研究的灵魂——市场不知道但你知道的东西。**

规则：
- **只写3–5个关键变量**（不能更多）
- 每个变量必须是**非共识的或被市场忽视的洞察**
- 禁止写"营收增长"、"利润率提升"等显而易见的废话
- 每个变量说明：（a）为什么重要？（b）当前市场如何定价这个变量？（c）你的判断与市场共识的差异是什么？

**示例（好）**：
> **关键变量2：负营运资本结构的隐性价值**
> 公司预收款/应收账款比 ≈ 4.2x，意味着客户实际在"免费借钱"给公司运营。
> 大多数分析师按账面ROE评估资本效率，但这一结构意味着实际投入资本比账目
> 低约30%，调整后真实ROIC比表观ROIC高约8–10pp。市场未充分定价这一结构性优势。

**示例（差，禁止）**：
> 公司有望实现收入增长和利润率改善，这将推动估值提升。（❌ 废话）

### 第五节：赔率评估（Odds & Payoffs）

**格式**：固定三情景概率分布 + 预期回报矩阵

**三情景概率设置规则**：
- 概率之和 = 100%（强制）
- 禁止使用等分概率（各33%），必须有主观判断
- 概率设置需要基于Task 4的三情景分析，不能凭空设定

```
┌─────────────────────────────────────────────────────────────────┐
│                    赔率评估矩阵                                    │
├────────────┬──────┬──────────┬──────────┬──────────────────────┤
│ 情景        │ 概率  │ 目标价（¥）│ 潜在回报  │ 驱动条件              │
├────────────┼──────┼──────────┼──────────┼──────────────────────┤
│ 悲观情景    │  [X%] │          │          │ 护城河比预期消退更快   │
│ 基准情景    │  [Y%] │          │          │ EPV + 部分GV实现       │
│ 乐观情景    │  [Z%] │          │          │ 护城河期超预期 + 增长   │
├────────────┼──────┼──────────┼──────────┼──────────────────────┤
│ 加权预期回报│      │          │ Σ(P×回报) │                      │
└────────────┴──────┴──────────┴──────────┴──────────────────────┘

概率设定理由：[50–100字简述为何选择这个概率分布，而非均等分布]
```

**赔率合理性检验**：
- 若加权预期回报 < 10%：重新检查是否有足够安全边际
- 若悲观情景也有正回报（绝对值）：强力买入信号
- 若悲观情景亏损 > 40%：需要评估仓位上限

### 第六节：买入/退出规则（Entry / Exit Rules）

**这是最让买方与卖方区别的部分——必须给出具体行动规则**

#### 6.1 买入规则

```
买入触发条件（满足全部才执行）：
  [ ] 条件1：价格低于 [¥/$ X]（= EPV基准 × (1 - 30%)）
  [ ] 条件2：近一季度业绩无重大负面信号（FCF/净利润 > 0.6）
  [ ] 条件3：没有Kill-Switch 红线被触发
  [ ] 条件4：[特异性条件，如护城河验证指标仍在预警阈值内]

仓位建议：
  初始仓位：[X%]（低于目标仓位，待观察）
  目标仓位：[Y%]（价格跌至Z时加仓至此）
  最大仓位：[W%]（不论价格多低均不超过此比例，控制个股风险）
```

#### 6.2 持仓管理 Tripwire 系统

**Tripwire条件**（任一触发则自动评估减仓/清仓）：

```
Tripwire A（护城河侵蚀）：
  触发：[具体指标]连续2季低于[阈值]
  行动：减仓50%，重新运行Task 4评估

Tripwire B（盈利质量恶化）：
  触发：FCF/净利润连续2季 < 0.5
  行动：Immediate — 全面复核EPV计算

Tripwire C（管理层信心信号）：
  触发：大股东/高管累计减持初始持股20%以上
  行动：启动完整重评估（从Task 1开始）

Tripwire D（行业外部冲击）：
  触发：[描述具体可能的外部事件]
  行动：[具体应对]
```

#### 6.3 退出规则

```
减仓节点：
  价格达到 EPV基准 × (1 + 20%) → 减仓至目标仓位50%
  价格达到 含GV基准估值 → 接近全清

清仓条件（任一）：
  [ ] 任意Tripwire红灯触发且重评估后判断无法修复
  [ ] 价格超越乐观情景估值（安全边际归零）
  [ ] Kill-Switch 被触发（任何任务阶段）
  [ ] [公司特异性：如核心产品失去牌照/ROIC突破性下降]
```

### 第七节：可证伪条件（Falsifiability）

**格式**：≤5条，每条必须包含"如果X发生 → 该判断Y失效"的可测量结构

**好的可证伪条件**：
> 可证伪条件1：若[公司]核心产品毛利率在下一财年低于X%（跌破5年历史低值），
> 则本分析"成本领先型护城河"的判断失效，EPV需下调20%以上，建议立即重评估。

**差的可证伪条件（禁止）**：
> 如果基本面恶化则需要重新评估。（❌ 不可测量）

五条可证伪条件必须分别对应：
1. 护城河判断
2. 盈利正常化假设
3. 成长价值（GV）假设（无GV时改为"EPV底线假设"）
4. 管理层资本配置质量判断
5. 行业竞争格局判断

---

### 第八节：未解问题（Open Questions）

**≤3项，聚焦"高不确定性 + 高价值"的信息缺口**

格式：
```
未解问题 1：[问题描述]
  为什么重要：[影响哪个估值维度，影响幅度估计X%]
  如何解答：[调研路径：拜访管理层 / 专家网络 / 一手数据采集]
  预计何时有答案：[时间判断]
  若不解答的保守处理：[默认假设是什么]
```

## 写作禁区

以下表达在本文档中绝对禁止：

| 禁止表达 | 原因 |
|---------|------|
| "综合来看"、"整体而言" | 回避结论的信号 |
| "视情况而定"、"不确定性较高" | 必须量化不确定性 |
| "有望实现"、"可能提升" | 用概率替代 |
| "基本面良好" | 无信息量的形容词 |
| "一定"、"肯定"、"必然" | 违反概率化原则 |
| 三情景各33%概率 | 必须有主观判断 |

## 最终质量标准

完成备忘录后，所有条件必须满足：

- [ ] 第一节≤3句话，结论立场明确（买入/观察/回避），含置信度标注
- [ ] 第二节：2个有效数字精度的每股估值，5个基准，安全边际%
- [ ] 第三节：护城河评级明确，最大威胁具体可识别
- [ ] 第四节：3–5个非共识的关键变量（无废话）
- [ ] 第五节：三情景概率之和=100%，有加权预期回报
- [ ] 第六节：买入价格触发点具体，Tripwire条件可测量
- [ ] 第七节：≤5条可证伪条件，均有可测量的触发事件
- [ ] 第八节：≤3个高价值未解问题，有解答路径
- [ ] 总字数2,000–3,000字（信息密度高，无填充内容）
- [ ] 所有数字与前序任务输出一致（AV/EPV/GV不得有出入）
