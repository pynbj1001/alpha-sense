# War-Game Workflow — L6 全局态势推演 × 论持久战式沙盘 v3.0

适用：`@L6 [公司]`、`@沙盘 [公司]`、`@推演 [公司]`、`@红队 [公司]`、`@war room [公司]`

⚠️ **强制前置条件**：
1. **L5 确认**：必须先完成 L5 备忘录。若用户跳过 L5，必须在报告开头打上红色警告。
2. **战场数据铁律**：严禁直接推演！必须先拉取标的及 1-3 家最核心竞争对手的实时财务/估值/市占率快照。

---

## 核心作战模块 (九大模块，缺一不可)

> 执行逻辑链：问题提起 → 相关方扫描 → 四维力量对比 → 行为驱动因素 → 主要矛盾 → 阶段推演 → 红蓝对抗 → 终局图景 → 末日协议 → 执行兵法

### 模块零：问题的提起
- 收集市场上的两种极端观点：“速胜论”（极度看多）与“亡国论”（极度看空）。
- 回顾 L5 结论，明确本次推演的核心问题。

### 模块一：全局相关方扫描 (6层穷举)
以标的为中心，扫描至少 15 个相关方：
1. **内部力量**：管理层、核心员工、股东。
2. **竞争力量**：直接竞对、潜在进入者、替代品。
3. **产业链**：上下游、渠道伙伴。
4. **资本市场**：机构、做空者、散户。
5. **监管与政策**：行业监管、宏观政策。
6. **社会与环境**：舆论、技术标准。
- **输出**：相关方阵营图谱及互动矩阵（标出谁可能变节）。

### 模块二：四维力量对比 (论持久战核心)
对标的与 1-3 家竞对进行四维横向对比：
1. **现实战力 (D1)**：营收、利润、现金、市占率（决定当前谁占上风）。
2. **发展趋势 (D2)**：商业模式是进步还是退步？技术路线是否顺应时代？（决定最终谁赢）。
3. **战略纵深 (D3)**：TAM天花板、资金储备、时间冗余（决定能撑多久）。
4. **外部支援 (D4)**：生态系统、用户忠诚度、政策支持（决定阵营强弱）。
- **态势结论**：指出“速胜论”和“亡国论”错在哪里，明确长期胜出的基础。

### 模块三：行为驱动因素拆解
针对前 5-10 个高影响力相关方，逐一拆解：
- 它们的利益函数（首要目标/底线红线）。
- 它们内部的主要矛盾。
- 它们的行为约束与惯性。

### 模块四：主要矛盾识别
从繁杂的线索中提炼出 **唯一一个** 左右全局的主要矛盾（如：增长vs盈利，创新vs守成）。
- 判定当前哪一方占支配地位？
- 预判：什么触发条件会导致矛盾翻转？

### 模块五：三阶段战局演化
判定标的处于：**战略防御期** / **战略相持期** / **战略反攻期**。
进行“如果…那么”链式模拟推演：
- **阶段一 (T+1年)**：当前行为惯性下的结果。
- **阶段二 (T+3年)**：力量此消彼长的转折点，最复杂的相持阶段。
- **阶段三 (T+5年)**：收敛至终局。

### 模块六：红蓝军深度对抗
强制左右互搏：
- **蓝军 (多方)**：基于趋势优势提出最强看多逻辑。
- **红军 (空方)**：基于现实劣势、资本错配或相关方叛变，进行摧毁式反驳。
- **交锋与仲裁**：经过至少 2 轮交锋，由第三方裁判定夺胜负手。

### 模块七：终局图景 (T+5)
- 描绘 5 年后行业的三种可能终态结构（垄断/寡头/碎片化）。
- 给出标的所处地位的概率分布（如：45%绝对龙头，35%平庸化，20%被淘汰）。

### 模块八：末日协议 (黑天鹅防线)
- 构建 2-3 个真正的“逻辑崩塌”级末日情景（如技术路线被证伪、集体叛变、监管封杀）。
- 设计具体的对冲方案。

### 模块九：执行兵法与 Tripwire 系统
- **作战方式**：明确当前应该打游击战（试探）、运动战（波段）还是阵地战（死守）。
- **建仓/加仓/减仓计划**：明确仓位上限。
- **Tripwire 系统 (绊线)**：强制输出包含正向、负向、黑天鹅的监控指标及触发阈值表格。
