# CIO-Memo Workflow — 买方 CIO 备忘录 v6.0

适用：`@L5 [公司]`、`@CIO备忘录 [公司]`

⚠️ **场景设定**：你的读者是管着几十亿资金的 CIO。他们没有时间看财报流水账，他们只关心：**这笔交易的胜率/赔率是多少？市场预期错在哪里？底线风险是什么？我该在什么位置扣动扳机？**

---

## 备忘录核心原则

1. **绝对精简 (Ruthless Brevity)**：不超过 1500 字。结论先行。
2. **聚焦非共识 (Focus on Variant Perception)**：市场都在说它好，你得说它哪里差；市场极度悲观，你得指出被错杀的隐蔽资产。
3. **可执行性 (Actionability)**：不要给"长期看好"这种废话，给出明确的建仓价位和仓位上限建议。

**search_web 非共识与风险搜索（硬性前置步骤）：**
- `search_web("[公司名] bear case risks [year]")` — 空头观点和最大风险
- `search_web("[公司名] bull case variant perception")` — 非共识多头论点
- `search_web("[公司名] latest news management")` — 管理层最新动态
- → 提取结果直接用于填充第 2 节（非共识洞察）和第 3 节（护城河与毁灭点）

---

## 备忘录结构 (固定模板，严禁篡改)

```markdown
# ⚡ 投资决策备忘录 | [公司名称] ([代码])
**日期：[YYYY-MM-DD] | 当前市值：[XX] | 当前价格：[XX]**

## 1. 核心定调 (The Verdict)
- **交易性质**：[困境反转 / 价值回归 / 优质复利 / 纯粹投机 / 价值陷阱]
- **决策建议**：[立即买入 / 触发条件买入 / 坚决回避 / 卖出融券]
- **期望赔率**：向下看底线估值跌幅 XX%，向上看合理估值涨幅 XX%，赔率比 [X:1]。

## 2. 非共识洞察 (Variant Perception)
- **市场共识是**：[描述目前主流卖方和散户的普遍看法]
- **我们的非共识是**：[指出市场错误定价的核心原因，例如错判了周期位置、忽略了隐性资产、高估了某项风险的持久性]
- **数据证明**：[给出 1~2 个最具杀伤力的硬数据]

## 3. 护城河与毁灭点 (Moat & The Kill-Switch)
- **保护我们的壁垒是**：[一句话说明最核心的竞争优势]
- **可能杀死这笔投资的黑天鹅是**：[指出最大的右侧尾部风险，绝不写"宏观经济下行"这种废话，要写具体的如"某某核心客户自研替代方案"]

## 4. 战术执行 (Execution Plan)
- **建仓区间**：≤ ¥/$ [价格] (对应 EPV 的 XX 折)
- **仓位建议**：建议配置上限 XX%
- **离场 Tripwire (绊线)**：一旦 [某具体指标/事件] 发生，不要犹豫，立刻清仓。
```
