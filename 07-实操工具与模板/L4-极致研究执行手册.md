# L4 极致研究 × Turn Every Page — 执行手册 v1.0

> **L3 回答"值不值得关注"，L4 回答"有没有任何死角是我们漏掉的"。**
> **L3 是分析师写研报，L4 是首席研究官事无巨细地翻完每一页。**

---

## 定位：为什么需要 L4？

| 级别 | 核心问题 | 思维模式 | 类比 |
|------|---------|---------|------|
| L3 | 这家公司好不好？值多少钱？ | **分析师** (Analyst) | 卖方出深度研报 |
| L4 | 有没有任何角落是我们没翻到的？ | **调查官** (Investigator) | 罗伯特·卡罗式的 Turn Every Page |

**核心区别**：
- L3 是**选择性覆盖** → "选 4-6 个最相关框架做深度分析"
- L4 是**全景式扫荡** → "8+ 框架全流派覆盖，不留死角，不留盲区"

**L3 的盲区，正是 L4 要填的**：
1. L3 选框架有偏见，优先调用分析师擅长的框架 → L4 **强制全流派穿透**
2. L3 聚焦公司本身 → L4 必须做**产业链上下游验证**和**跨市场估值锚**
3. L3 的竞争分析点到为止 → L4 要做**完整的竞争格局横向测评**
4. L3 的数据以公开报表为主 → L4 要追溯**产业专家、供应链、专利数据**等非常规维度
5. L3 是"够用就好"的实用主义 → L4 是"翻完每一页"的完美主义

|---------|------|
| `@L4 [公司名]` | `@L4 美团` |

**适用场景判断**（满足任一即触发 L4）：
- 拟配置 **>5% 组合权重**的重要持仓决策
- 首次进入**全新行业/全新市场**，需要系统性认知建立
- L3 分析中发现**多框架严重分歧**，需要全景验证
- 标的**商业模式复杂**，涉及多个业务分部、跨行业关联
- 用户明确要求"不留死角" / "翻遍每一页" / "百科全书式"分析

## L4 七大模块

### 模块一：预检深度审计 (Deep Pre-flight Audit)

> **原则：L3 的预检是 8 项快速检查，L4 的预检是 16 项深度审计。不仅要打勾，还要打分。**

#### 1.1 基础预检（继承 L3 的 8 项，升级为打分制）

| # | 检查项 | L3 要求 | L4 升级要求 | 评分(1-5) |
|:---:|---|---|---|:---:|
| 1 | 能力圈检查 | 可理解即通过 | **绘制商业模式画布 (Business Model Canvas)**，确认每个要素清晰 | |
| 2 | 数据可得性 | 有公开数据即通过 | **列出所有数据源清单**，评估每个源的可靠性、时效性、偏差风险 | |
| 3 | 生命周期判断 | 简单归类 | **S 曲线精确定位** + 历史类比（找到同行业/类似模式的历史先例） | |
| 4 | 周期位置判断 | 行业周期大致判断 | **三维周期叠加**：行业周期 × 信用周期 × 技术周期 | |
| 5 | 市场情绪快照 | 共识方向判断 | **量化情绪指标**：分析师评级分布、空头仓位、期权偏度(Skew)、散户论坛热度 | |
| 6 | 信息质量评估 | 一手/二手判断 | **信息地图绘制**：标注已知(Known)、可查(Knowable)、不可知(Unknowable)区域 | |
| 7 | 既往分析回顾 | 查看历史报告 | **先验贝叶斯基准**：以历史报告结论为先验，明确本次分析要更新什么 | |
| 8 | 时间框架对齐 | 确认短/中/长期 | **多时间框架同时分析**：短中长期各自独立出结论，标注适用时间窗口 | |

#### 1.2 L4 专属深度预检（8 项新增）

| # | 检查项 | 具体要求 |
|:---:|---|---|
| 9 | **利益冲突审查** | 识别潜在的信息偏差：公司IR叙事偏差、券商利益冲突、媒体偏向 |
| 10 | **学术/专利审查** | 核心技术是否有学术争议？专利保护期多久？有无挑战者？ |
| 11 | **监管风险扫描** | 所在市场的监管环境、合规风险、政策不确定性 |
| 12 | **ESG 快速扫描** | 环境/社会/治理风险是否可能成为估值折价因素？ |
| 13 | **关联方排查** | 股东关联交易、管理层同时兼任、母子公司利益输送 |
| 14 | **资本市场历史** | 历史融资/回购/分拆/并购记录，管理层的资本运作偏好 |
| 15 | **信息不对称评估** | 我们能获得的信息 vs 内部人能获得的信息，差距有多大？ |
| 16 | **类似公司失败案例** | 是否存在商业模式相似但最终失败的前车之鉴？ |

**评分规则**：
- 总分 80 分满分（16 项 × 5 分）
- **≥ 64 分**：直接进入 L4 分析
- **48-63 分**：标注风险项后进入，报告中着重论述薄弱环节
- **< 48 分**：建议降级为 L3，或明确标注"信息严重不足，结论可靠性受限"

### 模块二：全景数据采集 (Panoramic Data Collection)

> **原则：L3 是"够用就好"，L4 是"把能翻到的每一页都翻过来"。**

#### 2.1 五维数据采集矩阵

| 维度 | 数据类别 | 具体指标 | 工具/来源 |
|------|---------|---------|----------|
| **财务数据** | 三表完整数据 | 最近 5-10 年利润表、资产负债表、现金流量表 | yfinance / akshare / 年报 |
| | 杜邦分解 | ROE = 净利率 × 资产周转率 × 权益乘数（5 年趋势） | 手动计算 |
| | 边际指标 | MROIC（边际资本回报率）、增量 CapEx 回报率 | 手动计算 |
| | 正常化盈利 | 剔除一次性项目后的核心净利润（5 年平均） | 手动调整 |
| | 自由现金流 | FCF、FCF Yield、FCF 转化率（OCF/Net Income） | yfinance |
| **估值数据** | 绝对估值 | DCF（含敏感性）、EPV、资产重置价值 | 手动建模 |
| | 相对估值 | PE/PB/PS/EV-EBITDA 的历史分位数（5 年/10 年） | yfinance + 手动 |
| | 市赚率 | PR = PE / ROE | 手动计算 |
| | 跨市场锚 | 全球同行业可比公司估值对比 | yfinance 多标的 |
| **竞争数据** | 市占率 | 行业 Top 5 市占率及趋势 | 行业报告 / akshare |
| | 护城河评分 | 巴菲特五力 + 波特五力双重检验 | 框架分析 |
| | 竞争对手财务 | 至少 3 家可比公司的关键财务指标 | yfinance |
| **产业链数据** | 上游 | 关键供应商集中度、原材料价格趋势、替代可能性 | 行业数据 |
| | 下游 | 客户集中度、终端需求驱动因素、替代产品威胁 | 行业数据 |
| | 产业链价值分配 | 各环节毛利率分布、利润池位置 | 手动分析 |
| **非财务数据** | 管理层 | 任期、持股、薪酬结构、过往业绩记录 | 年报 / SEC |
| | 员工 | 员工人数增速、关键岗位流失率、Glassdoor 评分 | 公开数据 |
| | 专利/研发 | 专利数量及引用率、R&D/Revenue 比率趋势 | USPTO / 年报 |
| | 客户声音 | NPS 评分、App Store 评分、社交媒体舆情 | 公开数据 |

#### 2.2 竞争格局横向测评（L4 核心增量）

**必须覆盖行业 Top 3-5 竞争对手**，构建如下对比矩阵：

```markdown
## 竞争格局横向测评

| 指标 | [标的公司] | [竞争对手A] | [竞争对手B] | [竞争对手C] | 行业中位数 |
|------|-----------|------------|------------|------------|-----------|
| 营收规模 | | | | | |
| 营收增速(3Y CAGR) | | | | | |
| 毛利率 | | | | | |
| 净利率 | | | | | |
| ROE | | | | | |
| ROIC | | | | | |
| 负债率 | | | | | |
| PE | | | | | |
| PB | | | | | |
| FCF Yield | | | | | |
| 市占率 | | | | | |
| R&D/Revenue | | | | | |
| 股息率 | | | | | |
```

**横测后必须回答 3 个问题**：
1. 标的公司在哪些指标上**显著优于**行业？（护城河验证）
2. 标的公司在哪些指标上**显著落后**？（风险识别）
3. 估值溢价/折价是否与基本面差异**匹配**？（估值合理性）

#### 2.3 跨市场估值锚（L4 核心增量）

不同市场的同类公司估值对比（打破A股/港股/美股的估值幻觉）：

```markdown
## 跨市场估值锚

| 公司 | 市场 | PE | PB | PS | EV/EBITDA | 营收增速 | ROE | 备注 |
|------|------|-----|-----|-----|-----------|---------|------|------|
| [标的公司] | [市场] | | | | | | | 分析对象 |
| [全球可比A] | [市场] | | | | | | | |
| [全球可比B] | [市场] | | | | | | | |
| [全球可比C] | [市场] | | | | | | | |
| **中位数** | - | | | | | | | 估值锚 |
```

**结论格式**：标的公司相对全球可比中位数，溢价/折价 X%，这一溢价/折价是否合理？

### 模块三：全流派框架扫荡 (Full-Spectrum Framework Sweep)

> **原则：L3 选 4-6 个框架，L4 要求 8+ 个框架做到全流派穿透。不是为了堆砌，而是为了交叉验证的完备性。**

#### 3.1 九大维度强制覆盖

L4 的框架分析必须覆盖以下九大维度，每个维度至少用一个框架：

| 维度 | 核心框架 | 备选框架 | 输出要求 |
|------|---------|---------|---------|
| **① 估值** | 格林沃尔德三重估值 | 市赚率、DCF、可比估值 | 三重估值数值 + 估值区间 |
| **② 质量** | 巴菲特护城河 | 费雪 15 问、BG 韧性公司 | 护城河类型 + 耐久性评估 |
| **③ 成长** | 10X Alpha / 戴维斯双击 | 技术革命指数引擎 | 成长可持续性判断 + g 估计 |
| **④ 安全边际** | 卡拉曼安全边际 | 逆向投资、散户乙十年回本 | 下行风险量化 |
| **⑤ 周期** | 周金涛康波 / 三重共振 | 胡猛利润率体系 | 周期位置 + 利润率拐点 |
| **⑥ 管理层** | 费雪管理层评估 + BG 创始人 | 无限投资者框架 | 管理层评分 |
| **⑦ 商业模式** | 贝叶斯拐点 / 复利商业模式 | 禅道投资 | 模式可持续性 + 飞轮识别 |
| **⑧ 量化打分** | Q-G-P-R 百分制打分 | ResAlpha 残差法 | 综合分数 + 各维度分项 |
| **⑨ 本土化** | 方伟框架 / 归江框架 | 价值投资 3.0、林园 | A 股特色因素检验 |

**注意**：第⑨维度仅适用于 A 股/港股标的。美股标的可用帕伯莱/王川替代。

#### 3.2 框架反向筛选（必须执行）

在全流派扫荡前，先标注哪些框架**不适用**于本标的（参照 copilot-instructions.md 的反向筛选表），避免无效分析浪费精力。

```markdown
## 框架适用性筛选

| 框架 | 是否适用 | 理由 |
|------|:--------:|------|
| 格林沃尔德三重估值 | ✅/❌ | |
| 市赚率 | ✅/❌ | |
| 散户乙十年回本 | ✅/❌ | |
| 10X Alpha | ✅/❌ | |
| 周金涛康波 | ✅/❌ | |
| 技术革命指数引擎 | ✅/❌ | |
| 红利质量 | ✅/❌ | |
| （其他框架...） | | |
```

#### 3.3 每个框架的标准化输出（5 要素模板）

每个框架的分析输出**必须包含**以下 5 要素（与 copilot-instructions.md 第三步一致，但 L4 要求更详尽）：

```markdown
### [框架名称] 分析

**① 核心结论**（1 句话）：
**② 置信度**：XX% - YY%
**③ 关键假设**（最多 3 条，每条可证伪）：
  - 假设 1：...。证伪条件：如果 [X]，则该假设不成立。
  - 假设 2：...。证伪条件：...
  - 假设 3：...。证伪条件：...
**④ 二阶思考**：
  - 如果上述假设成立 → [连锁反应 1] → [连锁反应 2]
  - 线性 or 非线性？反身性？
**⑤ 反面论证**：最可能推翻本结论的证据是 [X]。
```

### 模块四：产业链纵深穿透 (Value Chain Deep Dive)

> **原则：L3 只看公司本身，L4 要穿透到供应商的供应商、客户的客户。**

#### 4.1 产业链地图绘制

```
上上游（原材料/核心零部件）
  ↓ [议价权: 强/弱] [替代性: 高/低]
上游供应商（直接供应商 Top 3-5）
  ↓ [成本占比: X%] [切换成本: 高/中/低]
────────────────────────────────────
★ 标的公司 [价值链位置] [利润池占比]
────────────────────────────────────
  ↓ [客户集中度: Top5占比X%] [切换成本: 高/中/低]
下游客户（直接客户 Top 3-5）
  ↓ [终端需求弹性: 高/低]
终端消费者/最终用户
```

#### 4.2 产业链风险扫描

| 风险类型 | 具体风险 | 概率 | 影响(1-5) | 预警信号 |
|---------|---------|:----:|:---------:|---------|
| **上游断供** | 关键原材料/芯片/IP 供应中断 | | | |
| **上游涨价** | 原材料/零部件成本上升侵蚀利润 | | | |
| **下游砍单** | 大客户缩减采购/转向竞品 | | | |
| **下游议价** | 客户集中导致压价 | | | |
| **替代技术** | 新技术路线淘汰当前方案 | | | |
| **产业政策** | 政府补贴/关税/环保法规变化 | | | |
| **垂直整合** | 上游或下游自研替代 | | | |

#### 4.3 利润池分析

量化产业链各环节的利润分配：

| 价值链环节 | 收入占比 | 毛利率 | 利润池占比 | 趋势 | 标的公司参与度 |
|-----------|---------|--------|-----------|------|-------------|
| 上游原材料 | X% | X% | X% | ↑/↓/→ | 无/部分/深度 |
| 中游制造 | X% | X% | X% | ↑/↓/→ | |
| 下游品牌/渠道 | X% | X% | X% | ↑/↓/→ | |
| 售后服务 | X% | X% | X% | ↑/↓/→ | |

**关键判断**：标的公司是否处于利润池最丰厚的环节？利润池正在向哪个方向迁移？

### 模块五：历史先例与类比分析 (Historical Analogues)

> **原则："历史不会重复，但会押韵。" L4 必须找到 2-3 个历史先例，检验投资论点的可靠性。**

#### 5.1 类比公司筛选

寻找满足以下条件之一的历史先例：
- 同行业、同发展阶段的前辈公司
- 类似商业模式但不同行业的类比公司
- 面临过类似竞争/周期/技术变革的公司

#### 5.2 先例分析模板

```markdown
### 历史先例 #[N]：[公司名] ([代码])

**类比维度**：[说明为什么选这个先例]

| 维度 | 先例公司（当时） | 标的公司（现在） | 相似度 |
|------|----------------|-----------------|:------:|
| 行业位置 | | | 高/中/低 |
| 发展阶段 | | | |
| 竞争格局 | | | |
| 盈利模式 | | | |
| 估值水平 | | | |
| 核心风险 | | | |

**先例结局**：[成功/失败/转型]
**对标的公司的启示**：
1. 如果历史重演 → [预期结果]
2. 与先例的关键差异 → [可能导致不同结局的因素]
3. **最重要的教训** → [一句话总结]
```

#### 5.3 统计基准率 (Base Rate)

回答以下问题：
- 该行业/该发展阶段的公司，5 年后仍是行业领导者的概率？（历史统计）
- 同类估值水平的公司，3 年后跑赢大盘的概率？
- 同类增长预期的公司，实际达标的概率？

> **"先看基准率，再看个案。" — 卡尼曼**

### 模块六：前瞻性分析增强版 (Enhanced Forward-Looking Analysis)

> **原则：L3 的前瞻性分析是标配 4 项，L4 在此基础上增加渗透率曲线、学习曲线、终局图景初判。**

#### 6.1 继承 L3 的四项前瞻分析

1. **领先指标映射**（与 L3 一致，但 L4 要求至少 3-5 个领先指标）
2. **催化剂时间表**（与 L3 一致，但 L4 要求覆盖未来 12-24 个月）
3. **情景分析**（四情景：牛市/基准/熊市/黑天鹅）
4. **二阶效应推演**

#### 6.2 L4 专属：渗透率/TAM 严格验证

> **警告：TAM 膨胀是最常见的分析陷阱。L4 必须严格验证。**

| 维度 | 问题 | 验证方法 |
|------|------|---------|
| **TAM 来源** | 引用的 TAM 数据是谁测算的？有无利益冲突？ | 比对至少 2 个独立来源 |
| **渗透率曲线** | 当前渗透率处于 S 曲线的什么位置？ | 绘制渗透率曲线 + 标注拐点 |
| **天花板检验** | 理论天花板 vs 实际天花板（支付意愿/能力约束） | 自上而下 + 自下而上交叉验证 |


## 质量检查清单

| # | 检查项 | 标准 |
|:---:|--------|------|

| 2 | **竞格横测完整性** | 是否包含 ≥3 家可比公司？数据是否为同期可比？ |
| 3 | **跨市场锚有效性** | 跨市场估值比较是否调整了增长率/风险差异？ |
| 4 | **产业链穿透深度** | 是否真正穿透到上下游？还是仅仅罗列了名称？ |
| 5 | **历史先例相关性** | 选取的先例是否真的与标的可比？有无"形似神不似"？ |
| 6 | **TAM 独立验证** | 是否对 TAM 做了独立的自上而下+自下而上验证？ |
