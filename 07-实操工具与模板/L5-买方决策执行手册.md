# L5 买方决策 × CIO 备忘录 — 执行手册 v2.0

> **L4 回答"有没有死角没覆盖"，L5 回答"核心变量是什么，赔率值不值得下注"。**
> **L4 是百科全书式的研究官，L5 是首席投资官写给合伙人的一页纸决策。**

---

## 定位：为什么需要 L5？

| 级别 | 核心问题 | 思维模式 | 类比 |
|------|---------|---------|------|
| L4 | 有什么是我遗漏的？ | **调查官** (Investigator) | 翻完每一页的百科全书 |
| L5 | 关键变量是什么？赔率如何？行动指令？ | **决策者** (Decision Maker) | CIO 向合伙人递交备忘录 |

**核心区别**：
- L4 是**做加法** → "尽可能多地覆盖信息、框架、维度"
- L5 是**做减法** → "从海量信息中蒸馏出 1-3 个 Key Variables，聚焦赔率和行动"

**L4 的盲区，正是 L5 要填的**：
1. L4 覆盖全面但**信息量过大**，决策者看完可能更迷茫 → L5 必须**化繁为简**
2. L4 做了多框架分析但没有做**框架融合** → L5 要求"无招胜有招"，打破框架边界
3. L4 给估值区间但没有做**单位经济学深度推演** → L5 必须构建 UE 模型
4. L4 列了风险但没有做**变量敏感性测试** → L5 必须量化"关键变量变化 1% 的影响"
5. L4 用行业公开数据 → L5 要挖掘**非共识数据**（市场忽略的隐形资产或风险）
6. L4 是"分析报告"的思路 → L5 是"投资备忘录"的思路：**说人话，给行动指令**

## ⚠️ Critical Constraints — 执行前必读

> **L5 是决策文件，不是分析报告。违反以下规则 → 备忘录失去决策价值，必须重写。**

### 1. 数据铁律（Decision-Grade Data）

**🚨 L5 的数据必须是当前实时数据。任何过时数据都会导致错误决策。**

```python
# ✅ L5 决策级数据获取（含价格、敏感性基准）
import yfinance as yf
from datetime import date

print(f"=== L5 决策数据快照 | {date.today()} ===")

ticker = yf.Ticker("SYMBOL")
info = ticker.info

# 决策关键数据
current_price = info.get("currentPrice") or info.get("regularMarketPrice")
market_cap    = info.get("marketCap", 0) / 1e9
pe            = info.get("trailingPE", "N/A")
roe           = info.get("returnOnEquity", 0)
rev_growth    = info.get("revenueGrowth", "N/A")
cash          = info.get("totalCash", 0) / 1e9
debt          = info.get("totalDebt", 0) / 1e9
net_debt      = debt - cash

print(f"当前价：${current_price:.2f} | 市值：${market_cap:.1f}B")
print(f"PE：{pe} | ROE：{roe:.1%} | 净负债：${net_debt:.1f}B")
print(f"营收增速(TTM)：{rev_growth}")
```

**L5 数据禁区**：
- ❌ 使用 L3/L4 报告中的数据而不重新获取（市场每天变化）
- ❌ 用"估计"或"约"描述当前股价、市值
- ❌ 敏感性矩阵用近似值而非实际计算结果填格子

### 2. 唯一交付物政策（Single Deliverable Policy）

L5 只产出 **一个文件**：

```
✅ 存入：10-研究报告输出/YYYY-MM-DD-个股-[公司名]-L5决策备忘录.md
```

**绝对禁止**：
- ❌ 输出"框架分析中间文件"（L5 直接产出备忘录）
- ❌ 单独生成"敏感性分析表格文件"
- ❌ 单独生成"行动指令清单"（行动指令在备忘录第 6 节内）
- ❌ "敬请阅读前置 L4 报告"的推诿输出（如无前置，内部快速执行 L3 扫描）

### 3. Key Variable 质量标准（KV 铁律）

L5 的核心是 KV，KV 质量决定备忘录的决策价值：

| KV 要求 | 验证标准 |
|---------|---------|
| **可观测** | 有具体的可查数据来源（不能是"管理层执行力提升"这类软性描述） |
| **可跟踪** | 有明确验证时间窗口（Q?/月?/年?) |
| **相互独立** | KV 之间皮尔逊相关系数 < 0.7（经验判断） |
| **数量上限** | 最多 3 个 KV，超过 3 个说明论点未蒸馏 |

**KV 示例对比**：
- ❌ 弱 KV：「管理层改善执行力」（不可测）
- ✅ 强 KV：「2026Q3 mRNA 肿瘤管线 Phase 3 数据读出，目标入组完成率 >90%」

### 4. 赔率与行动指令强制要求

- ✅ 上行/下行比（Upside/Downside Ratio）**必须用数字表示**（如 2.5:1）
- ✅ 概率加权目标价**必须给出具体数字**（如 $147）
- ✅ 行动指令**必须包含仓位比例和时间条件**（如"信号触发后 5 个交易日内，建仓至组合 3%"）
- ❌ 禁止用"可能适合买入"、"或许值得关注"等模糊表述

### 5. 执行时间参考

| 模块 | 预估时间 |
|------|---------|
| 数据获取（若无前置 L4） | 15–20 分钟 |
| 模块一 KV 蒸馏 | 15–20 分钟 |
| 模块二 UE 模型推演 | 20–30 分钟 |
| 模块三 敏感性矩阵 | 20–30 分钟 |
| 模块四 非共识挖掘 | 15–20 分钟 |
| 模块五 风控协议 | 10–15 分钟 |
| 模块六 行动指令 + 输出 | 15–20 分钟 |
| **全程合计（有前置 L4）** | **约 100–135 分钟** |
| **全程合计（无前置，内含 L3 扫描）** | **约 150–180 分钟** |

## L5 六大模块

### 模块一：L4 蒸馏 — 提取关键变量 (Distillation)

> **原则："你的报告很好，但用 30 秒告诉我，这个投资成不成，取决于什么？"**

#### 1.1 关键变量提取（最多 3 个）

从 L3/L4 的全景分析中，**蒸馏出 1-3 个 Key Variables**。这些是整个投资论点的"命门"：

```markdown
## 关键变量 (Key Variables)

### KV1：[变量名]
- **定义**：[用一句话解释这个变量]
- **当前状态**：[数值 / 定性描述]
- **我们的判断**：[预期方向 + 置信度]
- **市场共识**：[市场怎么看这个变量？]
- **预期差**：[我们与市场的分歧在哪里？差异有多大？]
- **验证时间**：[什么时候能验证？Q? / 月?]

### KV2：[变量名]
- （同上格式）

### KV3：[变量名]（如有）
- （同上格式）
```

**提取规则**：
- Key Variable 必须是**可观测、可跟踪**的变量（不是"管理层很优秀"这种含糊判断）
- 每个 KV 必须有明确的**验证时间窗口**
- KV 之间应尽量**相互独立**（非高度相关）
- 如果找不出 KV → 说明投资论点不够清晰，建议退回 L3 重新梳理

#### 1.2 一句话投资论题 (Investment Thesis Statement)

用**一句话**概括整个投资论题（强制字数限制：50 字以内）：

```
我们买入 [公司]，因为 [核心原因]，关键赌注是 [KV1]，
如果 [KV1 兑现]，预期回报 X%；如果 [KV1 落空]，最大损失 Y%。
```

> **"如果你不能用一句话说清楚为什么买入这只股票，你就不应该买。" — 芒格**

### 模块二：单位经济学深度推演 (Unit Economics Model)

> **原则："一笔生意赚不赚钱，不看总量，先看单位。" 宏大叙事之前，先拆最小经济单元。**

#### 2.1 识别最小经济单元

根据标的类型，选择合适的最小经济单元：

| 商业模式 | 最小经济单元 | 核心指标 |
|---------|------------|---------|
| **SaaS/订阅** | 单客户 | CAC、LTV、LTV/CAC、Payback Period、Churn Rate |
| **零售/餐饮** | 单店 | 单店营收、单店成本、单店利润、回本周期、坪效 |
| **制造业** | 单品/单线 | 单位成本、毛利、产能利用率、良品率 |
| **平台/电商** | 单用户/单交易 | ARPU、Take Rate、每笔订单毛利、获客成本 |
| **生物医药** | 单患者/单适应症 | ASP、COGS per patient、毛利/患者、可及患者数 |
| **金融** | 单客户/单产品 | 净利差、风险成本、手续费收入、capital efficiency |
| **广告/媒体** | 单用户 | ARPU、DAU/MAU、广告加载率、CPM |

#### 2.2 UE 模型模板

```markdown
## 单位经济学推演

### 最小经济单元：[定义]

| 关键变量 (Key Variable) | 数值 | 备注/来源 |
|:---|:---|:---|
| **ASP / 单价 / ARPU** | $[X] | [来源] |
| (-) COGS / 单位成本 | $[X] | [来源] |
| **= 单位毛利** | $[X] | **毛利率 [X]%** |
| (-) CAC / 获客成本 / 单店租金 | $[X] | [来源] |
| (-) 运营成本分摊 | $[X] | [来源] |
| **= 单位贡献利润** | $[X] | **贡献率 [X]%** |
| **回本周期** | [X] 个月 | |
| **LTV / 生命周期价值**（如适用） | $[X] | |
| **LTV / CAC**（如适用） | [X]x | 健康值 >3x |
```

#### 2.3 UE 规模效应验证

| 问题 | 回答 |
|------|------|
| 规模扩大时，单位成本是否下降？ | 是/否 + 幅度 |
| 规模扩大时，获客成本是否上升？ | 是/否 + 速度 |
| 单位经济学能否支撑盈利？规模多大时？ | 盈亏平衡产量/用户数 |
| 与竞争对手的 UE 比较如何？ | 优于/持平/落后 |

**L5 洞察格式**：
> **L5 Insight**: [用一句话指出市场对 UE 的误判。例如："市场高估了制造成本，低估了 mRNA 代码化生产带来的极高毛利率。"]

### 模块三：敏感性测试矩阵 (Sensitivity Matrix)

> **原则："关键变量变化 1% 时，你的估值变化多少？如果你回答不了，你就没有真正理解这个投资。"**

#### 3.1 单变量敏感性

对每个 Key Variable 做独立敏感性测试：

```markdown
## 单变量敏感性：[KV 名称]

| [KV 名称] | 悲观 (-2σ) | 偏悲观 (-1σ) | 基准 | 偏乐观 (+1σ) | 乐观 (+2σ) |
|:---|:---|:---|:---|:---|:---|
| **假设值** | [X] | [X] | [X] | [X] | [X] |
| **对应目标价** | $[X] | $[X] | $[X] | $[X] | $[X] |
| **隐含回报率** | [X]% | [X]% | [X]% | [X]% | [X]% |
```

#### 3.2 双变量敏感性矩阵（核心输出件）

选取**最重要的两个 Key Variables**，构建双变量矩阵：

```markdown
## 双变量敏感性矩阵

| [KV1] \ [KV2] | [KV2 悲观] | [KV2 偏悲观] | [KV2 基准] | [KV2 偏乐观] | [KV2 乐观] |
|:---|:---|:---|:---|:---|:---|
| **[KV1 悲观]** | $[X] | $[X] | $[X] | $[X] | $[X] |
| **[KV1 偏悲观]** | $[X] | $[X] | $[X] | $[X] | $[X] |
| **[KV1 基准]** | $[X] | $[X] | **$[X]** | $[X] | $[X] |
| **[KV1 偏乐观]** | $[X] | $[X] | $[X] | $[X] | $[X] |
| **[KV1 乐观]** | $[X] | $[X] | $[X] | $[X] | $[X] |
```

**矩阵解读要求**（必须回答）：
1. **安全边际**：当前股价处于矩阵的什么位置？有多少格子在当前价之上？
2. **赔率计算**：上行空间 / 下行空间 = ?（Upside/Downside Ratio）
3. **概率加权**：概率加权回报率（Expected Return）= Σ(概率 × 回报率)
4. **最大回撤**：最差情景下的最大亏损是多少？能否承受？

#### 3.3 情景概率分配

| 情景 | 概率 | 关键假设 | 目标价 | 回报率 |
|------|:----:|---------|:------:|:------:|
| **极乐观** | X% | [KV1 和 KV2 同时超预期] | $[X] | +X% |
| **乐观** | X% | [KV1 兑现] | $[X] | +X% |
| **基准** | X% | [维持现状] | $[X] | +X% |
| **悲观** | X% | [KV1 不达预期] | $[X] | -X% |
| **极悲观** | X% | [KV1 和 KV2 同时恶化] | $[X] | -X% |
| **概率加权** | 100% | — | **$[X]** | **+X%** |

**概率加权目标价与当前股价的关系**：
- 概率加权目标价 > 当前价 × 1.2 → **建议买入**（安全边际 > 20%）
- 概率加权目标价 ≈ 当前价 × 1.0-1.2 → **持有/观察**
- 概率加权目标价 < 当前价 → **建议回避**

### 模块四：非共识挖掘 (Non-Consensus Alpha)

> **原则："超额收益来自于你知道而市场不知道的东西，或者你理解而市场误解的东西。"**

#### 4.1 共识地图

先绘制市场的共识认知地图，然后逐项挑战：

```markdown
## 市场共识 vs 我们的判断

| 维度 | 市场共识 | 我们的判断 | 分歧程度 | 验证方式 |
|------|---------|-----------|:--------:|---------|
| 增长率 | [X]% | [Y]% | 大/中/小 | 季报/月度数据 |
| 利润率 | [X]% | [Y]% | | |
| 竞争格局 | [共识观点] | [我们的观点] | | |
| 估值合理性 | [市场定价隐含] | [我们的判断] | | |
| 管理层能力 | [市场评价] | [我们的评估] | | |
| 核心风险 | [市场担忧] | [我们的风险排序] | | |
```

#### 4.2 非共识信号挖掘

系统性地搜索市场可能忽略的信息：

| 非共识来源 | 具体信号 | 潜在影响 | 置信度 |
|-----------|---------|---------|:------:|
| **隐形资产** | 未体现在账面上的品牌/IP/数据/牌照价值 | | |
| **隐形负债** | 表外风险、或有负债、诉讼风险 | | |
| **被忽视的管线/业务线** | 市场未定价的增长期权 | | |
| **供应链信号** | 供应商/客户的异常行为 | | |
| **内部人行为** | 管理层/大股东的买卖信号 | | |
| **跨市场信号** | 其他市场的类似标的走势暗示 | | |
| **学术/技术前沿** | 最新论文/专利暗示的趋势变化 | | |

#### 4.3 Alpha 来源归因

回答核心问题：**如果这个投资赚钱了，超额收益到底来自哪里？**

| Alpha 来源 | 描述 | 持续性 |
|-----------|------|:------:|
| **信息差** | 我们掌握了市场未知的信息？ | 短期 |
| **理解差** | 我们对同一信息的理解更深？ | 中期 |
| **时间差** | 我们的时间框架更长，能承受波动？ | 长期 |
| **行为差** | 市场因情绪/制度原因错误定价？ | 中期 |
| **结构差** | 指数/ETF/被动投资造成的结构性错配？ | 中长期 |

> **最有价值的 Alpha**：理解差 > 时间差 > 行为差 > 信息差 > 结构差

### 模块五：Kill-Switch 增强 + 刚性风控 (Enhanced Risk Protocol)

> **原则：L3/L4 的 Kill-Switch 是"检查"，L5 的是"执法"。不是打勾了事，而是设定具体数值、预设行动。**

#### 5.1 Kill-Switch 刚性检查（含量化阈值）

| Kill-Switch | 检查结果 | 量化数据 | 阈值 | 判定 |
|---|:---:|---|---|:---:|
| **数据缺失** | PASS/FAIL | 数据完整度: X/10 | ≥7 | |
| **能力圈越界** | PASS/FAIL | 商业模式一句话解释 | 能/不能 | |
| **管理层红旗** | PASS/FAIL | 关联交易金额/营收: X% | <5% | |
| **杠杆红线** | PASS/FAIL | 有息负债/EBITDA: Xx | <4x | |
| **安全边际** | PASS/FAIL | 当前价/EPV: X% | >20%边际 | |
| **单一依赖** | PASS/WARN | Top 客户占比: X% | <50% | |
| **全面共识** | PASS/WARN | 看多分析师占比: X% | <80% | |

#### 5.2 下行保护评估（L5 核心增量）

回答一个直击灵魂的问题：**如果我完全错了，最大亏多少？**

```markdown
## 下行保护评估

### 硬底计算
| 下行支撑 | 数值 | 隐含最大跌幅 | 说明 |
|---------|------|:----------:|------|
| **清算价值** | $[X]/股 | -X% | 假设破产清算 |
```

## L5 写作风格指南

### 核心原则：深入浅出（"说人话"）

| 要素 | L4 风格 | L5 风格 |
|------|--------|--------|
| **语言** | 学术/专业 | 对话/直白 |
| **数据** | 尽可能详尽 | 只呈现关键数据 |
| **图表** | 多多益善 | 每张图表必须有"So What" |
| **结论** | 多角度分析 | 旗帜鲜明 + 行动指令 |
| **风险** | 全面列举 | 按杀伤力排序，聚焦前 3 个 |
| **篇幅** | 不限 | 控制在 2000-4000 字（精炼） |

### 写作要求

1. **每段必须有观点** — 禁止纯粹的事实罗列段
2. **每张表必须有解读** — 数据后面必须跟"So What"
3. **用类比解释复杂概念** — "它就像是 XX 行业的 YY"
4. **旗帜鲜明** — 不用"可能"、"也许"、"有待观察"等含糊表达（概率化表达除外）
5. **结尾必须有行动指令** — "所以我们应该..."

## 质量检查清单（报告提交前必过 12 关）

> **"CIO 拿到这份备忘录，30 秒内能做出决策——这是 L5 的唯一合格标准。"**
> **以下每一项未通过 → 修改后再提交，不能跳过。**

### A. 数据质量（Decision-Grade Data）

- [ ] **实时数据核实**：报告包含运行日期的数据快照（`date.today()` 输出或说明数据截止日）
- [ ] **敏感性矩阵有真实数字**：双变量矩阵的每个格子均有具体股价数值，非占位符 `$[X]`
- [ ] **概率加权目标价可计算**：情景概率分配中概率之和 = 100%，加权目标价已计算

### B. KV 质量（Key Variable Quality）

- [ ] **KV 数量合规**：1–3 个 KV，不超过 3 个
- [ ] **KV 可观测**：每个 KV 有明确的可查数据来源（无软性描述如"管理层改善"）
- [ ] **KV 有验证时间**：每个 KV 标注验证时间窗口（如 2026Q3 财报）
- [ ] **KV 独立性**：任意两个 KV 之间不高度相关（非同一宏观因素的两种表达）

### C. 决策质量（Decision Quality）

- [ ] **赔率数字化**：上行/下行比用具体数字表示（如 2.5:1），不是"上行空间更大"
- [ ] **行动指令可执行**：行动指令包含仓位比例（组合X%）、时间条件（信号后X周内）
- [ ] **下行保护诚实**：下行保护评估中，最大亏损估计无粉饰（取最保守的硬底计算）
- [ ] **Scenario Kill-Switch 预设**：每个关键情景已绑定预设行动，不是事后临时决策

### D. 输出格式（Output Format）

- [ ] **字数范围合规**：2000–4000 字（精炼，非L4百科全书）
- [ ] **文件命名正确**：`YYYY-MM-DD-个股-[公司名]-L5决策备忘录.md`
- [ ] **存档路径正确**：`10-研究报告输出/`
- [ ] **决策摘要完整**：摘要包含一句话论题 + 评级 + 置信度 + 目标仓位 + 赔率速览

### E. "30秒 CIO 测试"

> 把决策摘要给一位经验丰富的 CIO 看 30 秒，他/她能回答：
> - "这个投资的关键赌注是什么？"
> - "上行/下行各多少，概率各多少？"
> - "如果做，买多少仓位？"
> - "什么情况下必须止损？"
>
> 四问都能立即回答 → L5 合格。  
> 有一问含糊 → 找到对应章节（KV/赔率/仓位/Kill-Switch），强化后重提。
