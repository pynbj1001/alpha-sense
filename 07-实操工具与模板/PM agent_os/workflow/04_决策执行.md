# Phase 4: 决策与执行

## 触发条件
用户说 **"想买"**、**"想卖"**、**"下单"**、**"想做XX"**

## 核心原则

> 每一次可能进入交易的行为，都必须先生成 Decision Brief。
> Decision Brief 不是交易指令，而是一份判断说明。
> 它强制你把想法写清楚——写不清楚的不能做。

## 生成 Decision Brief

### Step 1: 关联判断
- 这笔交易基于哪个判断？
- 如果没有对应的判断 → 必须先建立判断（回到 Phase 2）
- 如果关联判断已被证伪 → 拒绝创建决策

### Step 2: 收集交易参数
| 参数 | 说明 | 是否必填 |
|------|------|---------|
| entry_price | 入场价格 | 建议填写 |
| stop_loss | 止损价格 | **必填** |
| take_profit | 止盈价格 | 建议填写 |
| position_size_pct | 仓位占比 | 必填 |

### Step 3: AI 审查决策
系统必须评估：
1. **逻辑清晰度** — 交易理由是否含混？如果含混，追问
2. **置信度诚实度** — 不允许模糊的"看好"或"看空"
3. **风险量化** — 止损位和仓位是否合理
4. **对齐判断** — 交易方向是否与判断一致
5. **情绪检测** — 决策理由是否充分，还是冲动驱动

### Step 4: 风控审查（→ Phase 5）
必须通过风控审查后才能执行。

## 确认流程

### 人工确认
用户说 **"确认 d001"** 时：
- 标记 decision 状态为 approved
- 记录执行时间
- 更新 `state/decisions.json` 和 `state/trades.json`
- 更新 `state/portfolio.json`

### 拒绝
用户说 **"拒绝 d001"** / **"取消 d001"** 时：
- 标记 decision 状态为 rejected
- 记录拒绝原因

### 强制执行
如果风控未通过但用户坚持，用户说 **"强制执行 d001"**：
- 记录这是一次强制执行（在每日复盘中会被重点审查）
- 标记为 force_executed
- 必须让用户写下原因

## 平仓
用户说 **"平仓 t001"** / **"卖出 t001"** 时：
- 记录平仓价格和时间
- 计算盈亏
- 更新 `state/trades.json`、`state/portfolio.json`
- 输出盈亏摘要

```
📤 已平仓 | 贵州茅台
  入场: 1650 → 出场: 1720
  盈亏: +7000 (+4.2%)
  持有时间: 2小时30分
  原因: 止盈目标接近 / 判断松动 / 手动平仓
```

## 状态文件更新
- `state/decisions.json` — 添加或更新决策
- `state/trades.json` — 记录交易
- `state/portfolio.json` — 更新持仓和资金
