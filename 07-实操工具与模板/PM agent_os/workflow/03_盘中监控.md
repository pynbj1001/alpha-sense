# Phase 3: 盘中监控

## 触发条件
用户说 **"更新"**、**"盘中"**、**"现在怎么样"**
或用户提供了最新的盘面信息

## 核心问题

> 盘中只需要回答一个问题：
> **「早上的判断，还站得住吗？」**

价格涨了 ≠ 判断正确
价格跌了 ≠ 判断错误
真正危险的是：判断的逻辑基础已经出现松动，但被价格波动的噪音所掩盖。

## 执行步骤

### Step 1: 读取当前状态
- `state/judgments.json` — 活跃判断列表
- `state/daily_boundary.json` — 今日边界条件
- `state/portfolio.json` — 当前持仓
- `state/trades.json` — 今日已执行交易

### Step 2: 判断状态检查
对每个活跃/松动判断，评估：
1. 判断所依赖的关键变量，结合用户提供的最新信息，是否出现变化？
2. 是否已经触发了任何证伪条件？
3. 当前市场走势是否与判断预期一致？
4. 是否有新的信息需要加入考量？

### Step 3: 边界条件检查
检查今日设定的边界条件是否被触碰：
- 大盘是否异常？
- 交易频率是否过高？
- 当日亏损是否接近阈值？

### Step 4: 行为偏差检测
如果用户提到想做什么操作，检测：
- 想交易的标的是否在判断列表中？
- 方向是否与判断一致？
- 是否在边界条件触发时仍想交易？
- 是否有情绪驱动的迹象？（急切、犹豫、报复性交易）

## 输出格式

```
──────────────────────────────────────────────────
  🔍 盘中监控报告 | HH:MM:SS
──────────────────────────────────────────────────
  风险等级: 🟢 LOW / 🟡 MEDIUM / 🟠 HIGH / 🔴 CRITICAL
  
  （一句话总结当前状态）

  ✅ [j001] 贵州茅台 — intact
      证据: 盘中走势符合预期...
      建议: 继续持有
  
  ⚡ [j002] 宁德时代 — weakening
      证据: 锂价继续下跌...
      建议: 考虑减仓

  🚫 边界条件检查:
      • 大盘状态: 正常
      • 今日交易次数: 1/5
      • 今日亏损: -0.5%（阈值 -3%）

  ⚠ 未确认告警: 2 条
      [a001] 判断 j002 出现松动
      [a002] 今日交易频率偏高
──────────────────────────────────────────────────
```

## 行为偏差检测报告
当用户提到想要操作时：

```
──────────────────────────────────────────────────
  🔍 行为偏差检测结果
──────────────────────────────────────────────────
  ⚠ 检测到偏差! 严重度: 70%
  类型: 计划外交易
  说明: 你想交易的标的不在今日判断列表中...
  
  ❓ 系统提问: 
  「你现在做的，还是你早上说的那件事吗？
    如果这是一个新的机会，请先建立判断。」
──────────────────────────────────────────────────
```

## 告警类型
| 告警类型 | 触发条件 | 严重度 |
|---------|---------|--------|
| judgment_weakened | 判断出现松动迹象 | MEDIUM |
| judgment_invalidated | 判断被证伪 | HIGH |
| frequency_anomaly | 短时间内交易次数异常 | MEDIUM |
| drawdown_warning | 当日亏损接近阈值 | HIGH |
| position_limit | 仓位接近上限 | MEDIUM |
| deviation_detected | 行为偏离计划 | HIGH |
| boundary_violation | 边界条件被触碰 | CRITICAL |
