# Phase 1: 盘前研判

## 触发条件
用户说 **"开盘"** 或 **"盘前"**

## 执行步骤

### Step 1: 加载系统状态
读取以下文件：
- `state/judgments.json` — 存量判断
- `state/portfolio.json` — 当前持仓
- `vault/` 最近的复盘记录 — 昨日教训

### Step 2: 市场环境分析
基于你的训练知识，分析：
1. **隔夜外盘**：美股、港股、欧洲的涨跌及原因
2. **大宗商品**：原油、黄金、铜等关键品种
3. **汇率**：美元指数、人民币走势
4. **宏观变量**：近期发布的重要经济数据
5. **政策动向**：央行、监管层近期的信号
6. **地缘政治**：影响市场的国际事件

如果用户提供了具体的市场数据或新闻，优先使用用户提供的信息。

### Step 3: 存量判断检查
逐一审查 `state/judgments.json` 中状态为 active 或 weakened 的判断：
- 判断所依赖的关键变量是否仍然成立？
- 是否出现了证伪条件？
- 置信度是否应该调整？
- 状态是否需要变更？（active → weakened → invalidated）

### Step 4: 设定今日边界条件
明确今日的"不出手"条件，例如：
- 大盘异常波动时不交易
- 情绪不佳时不交易
- 已亏损达到阈值时停止
- 基于昨日教训设定的特别约束

### Step 5: 输出晨间研判简报

## 状态文件更新
完成后需要更新：
- `state/daily_boundary.json` — 写入今日边界条件
- `state/judgments.json` — 如有判断状态变更，更新对应判断
