# L3 深度研报 × 质效双优 — 执行手册 v1.0

> **L2 回答"这个指标/维度怎么看"，L3 回答"这家公司到底好不好，值不值这个价"。**
> **L2 是单项分析，L3 是卖方首席分析师出具的完整深度研报。**

---

## 定位：为什么需要 L3 执行手册？

| 级别 | 核心问题 | 思维模式 | 类比 |
|------|---------|---------|------|
| L2 | 这个维度怎么看？ | **专项研究员** (Specialist) | 估值/护城河/拐点的单项答问 |
| L3 | 这家公司好不好？值多少钱？该不该关注？ | **首席分析师** (Lead Analyst) | 卖方深度研报 — 结论清晰、逻辑闭环 |
| L4 | 有没有任何死角我们漏掉了？ | **调查官** (Investigator) | Turn Every Page 百科全书 |

### L3 的核心定位：**质效双优**

- L2 偏"回答问题" → L3 是**主动构建论点**，输出完整投资观点
- L4 追求"无死角" → L3 追求**最高投入产出比**，用 4-6 个最精准的框架回答核心问题
- L3 是本知识库**使用频率最高**的分析级别（`@分析` 指令的默认级别）
- L3 的目标：**一篇报告读完，投资者能做出"关注 / 不关注 / 深入研究"的决策**

### L3 对 L2 的升级

| 维度 | L2 | L3 |
|------|-----|-----|
| 框架数量 | 2-3 个 | **4-6 个** |
| 数据覆盖 | 关键指标 + 历史分位 | **完整三表 + 行业对标** |
| 输出形式 | 结构化回答（对话内） | **完整报告，存入 10-研究报告输出/** |
| 前瞻分析 | 可选 | **必做**（情景分析 + 催化剂） |
| 交叉验证 | 可选 | **必做**（至少价值×成长二维） |
| Kill-Switch | 不要求 | **必做**（快速检查） |
| Q-G-P-R 打分 | 不要求 | **必做** |

### L3 相比 L4 的效率边界（何时止步于 L3）

| 维度 | L3 够了 | 需要升级 L4 |
|------|---------|-------------|
| 框架覆盖 | 4-6 个最相关框架 | 需要全流派 8+ 框架穿越 |
| 竞争分析 | 行业格局概览 + 前2-3名对标 | 需要完整竞争横向测评 |
| 产业链 | 上下游概览 | 需要逐层验证供应链 |
| 数据源 | 公开报表 + yfinance/akshare | 需要专利、学术、产业专家等维度 |
| 时间投入 | 高效聚焦核心 | 不计成本翻完每一页 |
| 适用权重 | 跟踪池 / <5% 仓位 | >5% 重仓决策 |

**经验法则**：如果 L3 分析中发现**多框架严重分歧**——比如估值框架看便宜但成长框架看差——这是升级至 L4 全景验证的信号。

## L3 六大模块

### 模块一：高效预检 (Efficient Pre-flight)

> **原则：L4 预检是 16 项深度审计，L3 预检是 8 项快速检查 — 目标是 3 分钟完成，排除明显的"不该碰"，而非追求面面俱到。**

#### 1.1 八项预检清单

| # | 检查项 | L3 要求 | 判定标准 | 通过/预警/否决 |
|:---:|---|---|---|:---:|
| 1 | **能力圈检查** | 商业模式能否用一句话向外行说清？ | 说不清 → ⚠️ 预警，考虑能力圈外降级 | |
| 2 | **数据可得性** | yfinance/akshare 能否获取核心财务数据？ | 核心数据缺失 → ❌ Kill | |
| 3 | **生命周期判断** | 初创 / 成长 / 成熟 / 衰退，一句话归类 | 决定框架权重分配 | |
| 4 | **周期位置判断** | 行业是顺周期还是逆周期？大致位置？ | 影响估值方法选择 | |
| 5 | **市场情绪快照** | 当前共识看多/看空/分歧？有无极端情绪？ | 记录，作为逆向检查基准 | |
| 6 | **信息质量评估** | 研究资料充足度 1-5 分快速打分 | ≤2 分 → 预警，标注信息盲区 | |
| 7 | **既往分析** | `10-研究报告输出/` 有无历史报告？ | 有 → 作为先验基准 | |
| 8 | **时间框架** | 用户关心短期/中期/长期？默认中期 1-3 年 | 对齐后续估值与情景分析 | |

**预检速判规则**：
- 出现 1 个 ❌ → 停止分析，告知用户原因
- 出现 2+ 个 ⚠️ → 在报告开头标注风险提示，分析继续但降低置信度
- 全部通过 → 正常推进

**效率要求**：预检阶段**不做深度研究**，目的是快速排雷、确定框架方向。

### 模块二：框架智选 (Smart Framework Selection)

> **原则：L3 的灵魂是"选对框架"。不是框架越多越好，而是 4-6 个精准命中核心问题。**

#### 2.1 框架选取三步决策引擎

**Step 1**：判断分析情景（S1-S5）

| 情景 | 默认核心框架（必选 3 个） |
|------|--------------------------|
| S1 个股深度 | 格林沃尔德三重估值 + 护城河 + Q-G-P-R 打分 |
| S2 行业格局 | 三重共振 + 波特五力 + 渗透率/天花板 |
| S3 宏观周期 | 周金涛康波 + 李蓓宏观对冲 + 债券框架 |
| S4 投资决策 | 卡拉曼安全边际 + 市赚率 + 催化剂清单 |
| S5 十倍筛选 | 10X Alpha + 技术革命指数引擎 + 贝叶斯拐点 |

**Step 2**：根据标的属性叠加 1-3 个补充框架

| 标的属性 | 叠加框架（选 1-2 个最相关的） |
|----------|-------------------------------|
| 成熟期（ROE>15%, g<15%） | 市赚率 / 散户乙十年回本 |
| 高成长（g>25%） | 10X Alpha / 贝叶斯拐点 |
| 周期性 | 利润率体系（胡猛）/ 三重共振 |
| 平台/生态型 | BG 韧性公司 / 巴菲特护城河加权 |
| 困境反转 | 卡拉曼安全边际 / 贝叶斯拐点 |
| 科技颠覆 | 技术革命指数引擎 / ResAlpha |
| 垄断消费品 | 林园 / 散户乙 |
| A 股特色 | 方伟框架 / 归江框架 |

**Step 3**：反向筛选 — 剔除不适用框架

执行 `copilot-instructions.md` 中的反向筛选表，确认无误后锁定 **最终 4-6 个框架**。

#### 2.2 框架确认输出

在报告"分析框架"行写明：

```
分析框架：[框架1] × [框架2] × [框架3] × [框架4]（× [框架5]）
框架选择理由：[一句话说明为什么是这个组合]
主动排除：[被反向筛选剔除的框架及原因]
```

**效率护栏**：
- **严格控制 4-6 个框架**。超过 6 个 → 自问"是否应该升级为 L4？"
- 如果 3 个框架就够了 → 不必凑数，但在报告中说明"3 个框架已足够覆盖核心问题"
- **每个框架必须有独立的分析增量**。如果两个框架产出高度重叠，删掉一个

### 模块三：数据先行 (Data First)

> **铁律：先获取数据，再打开框架。绝不凭记忆做分析。**

#### 3.1 数据获取清单（L3 标准版）

| 数据类别 | 必获取指标 | 工具 | 验证要求 |
|---------|-----------|------|---------|
| **估值** | PE / PB / PS / EV-EBITDA / 市赚率PR | yfinance | PR = PE / ROE 手动计算验证 |
| **质量** | ROE / ROIC / 毛利率 / 净利率 / FCF | yfinance + 三表推算 | FCF = 经营现金流 - CapEx |
| **成长** | 营收增速 / 利润增速 / g（可持续增长率） | yfinance 历史三表 | 至少 3 年趋势 |
| **资本结构** | 负债率 / 有息负债 / Net Debt/EBITDA | yfinance 资产负债表 | — |
| **股东回报** | 股息率 / 回购率 | yfinance info | 可选：留存收益复合回报率 |
| **行业对标** | 前 2-3 名可比公司的 PE/ROE/g | yfinance 多标的 | 至少 2 家对标 |

**A 股标的额外使用 akshare**：
```python
import akshare as ak
df = ak.stock_financial_analysis_indicator(symbol="600519", start_year="2020")
```

#### 3.2 数据快照表模板

每份 L3 报告**必须**包含以下格式的数据快照表：

```markdown
## 数据快照表

> 数据获取日期：YYYY-MM-DD | 来源：yfinance / akshare

### 估值指标
| 指标 | 数值 | 行业中位数 | 历史分位 | 备注 |
|------|------|-----------|---------|------|

### 质量指标
| 指标 | 最近FY | 前一FY | 趋势 | 备注 |
|------|-------|--------|------|------|

### 成长指标
| 指标 | 数值 | 3年CAGR | 行业增速 | 备注 |
|------|------|---------|---------|------|

### 资本结构
| 指标 | 数值 | 趋势 | 备注 |
|------|------|------|------|
```

#### 3.3 合理性检查（4 项快速校验）

| 校验项 | 检查方法 | 异常处理 |
|--------|---------|---------|
| PE 合理区间 | 与行业中位数对比，偏差 > 50% 需解释 | 在报告中标注原因 |
| ROE 一致性 | 与过去 3 年趋势对比 | 大幅波动需用 DuPont 分解 |
| 营收 vs 行业 | 公司增速 vs 行业增速 | 差异大需解释份额变化 |
| 市值校验 | 股价 × 总股数 ≈ 报告市值 | 差异 > 5% 需排查 |

**效率护栏**：
- L3 数据获取**一次性完成**，双源交叉验证聚焦于 **PE 和 ROE 两个核心指标**（不必像 L4 那样每项都双源）
- 行业对标**选 2-3 家最直接的可比公司**，不做全行业横扫
- 数据获取 + 快照表构建**目标时间 10-15 分钟**

### 模块四：多框架并行分析 (Multi-Framework Analysis)

> **原则：从价值 × 成长两个维度交叉验证。宏观维度在 L3 中作为"背景检查"而非独立分析。**

#### 4.1 分析维度架构

```
                    ┌──────────────────────┐
                    │   L3 多框架分析架构    │
                    └──────────┬───────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
         A. 价值维度       B. 成长维度     C. 宏观/背景
        "值多少钱"        "能长多大"       "环境如何"
        ──────────        ──────────       ──────────
        2-3 个框架        1-2 个框架       快速定性判断
        ──────────        ──────────       ──────────
        深度执行           深度执行         概览即可
```

#### 4.2 A. 价值维度（必选 2-3 个框架，深度执行）

从以下菜单中选择 **2-3 个最适配**的框架：

| 框架 | 适用标的 | 核心输出 |
|------|---------|---------|
| 格林沃尔德三重估值 | 有稳定盈利的成熟企业 | 资产价值 → EPV → 成长价值的递进估值 |
| 巴菲特护城河 | 有竞争优势的质量公司 | 护城河类型 + 强度评分 |
| 卡拉曼安全边际 | 困境反转 / 低估标的 | 安全边际 % + 下行保护 |
| 市赚率 PR | ROE>8% 的成长企业 | PR 值 + 历史分位 + 合理PE区间 |
| 散户乙十年回本 | 高股息 + 稳定盈利 | 持有N年本金回收率 |
| Q-G-P-R 打分 | **所有标的（必选）** | 质量/成长/价格/风险四维百分制 |

> **Q-G-P-R 打分在 L3 中为必选项**，作为量化综合工具。

#### 4.3 B. 成长维度（选 1-2 个框架，深度执行）

| 框架 | 适用标的 | 核心输出 |
|------|---------|---------|
| 10X Alpha | 高成长 / 十倍股候选 | 十倍得分 + 成长持续性 |
| 贝叶斯拐点 | 不确定性高 / 拐点型 | 概率更新 + 置信度追踪 |
| BG 韧性公司 | 超长期持有候选 | 韧性评分 + 创始人/文化 |
| 费雪 15 问 | 成长质量验证 | 15 问逐项回答 |
| 戴维斯双击 | 利润 + 估值双升可能 | 双击概率 + 触发条件 |
| 技术革命指数引擎 | 科技/颠覆型 | 指数增长验证 |

#### 4.4 C. 宏观/背景维度（快速定性，非独立框架分析）

L3 中宏观维度**不做独立的完整分析**（那是 L4 的任务），而是作为**背景检查**：

```markdown
### 宏观 / 周期环境快照
- **宏观处于什么周期？** [扩张 / 见顶 / 收缩 / 复苏]
- **利率环境对标的的影响？** [利好 / 中性 / 利空]
- **行业周期位置？** [和标的分析相关的周期判断]
- **是否存在重大宏观风险？** [贸易/地缘/监管等]
```

#### 4.5 每个框架的标准输出格式

> **L3 中每个框架的分析必须包含以下 5 项，缺一不可。**

```markdown
### [框架名称]

**核心结论**：[一句话]

**置信度**：XX-XX%

**关键假设**（最多 3 条，每条可证伪）：
1. [假设 1] → 证伪条件：[如果 X 发生则判断错误]
2. [假设 2] → 证伪条件：[...]
3. [假设 3] → 证伪条件：[...]

**二阶思考**：如果上述假设成立 → [后续连锁反应是什么？]

**反面论证**：最可能推翻本结论的证据是 → [...]
```

**效率护栏**：
- 每个框架分析**控制在 300-500 字**（含表格）。超过 500 字的单框架 → 自问"是否过度深挖了？"
- 如果某框架不产生独立增量（和另一框架结论完全一致）→ **合并或删除**
- 宏观背景检查**控制在 100 字以内**，不展开

### 模块五：前瞻 + 综合 (Forward-Looking & Synthesis)

> **原则：L3 的前瞻性分析做"标准版"——情景分析必做，催化剂和领先指标精简版。**

#### 5.1 情景分析（必做）

| 情景 | 概率 | 关键假设 | 目标价/回报率 | 触发信号 |
|------|------|---------|-------------|---------|
| **牛市情景** | X% | [核心乐观假设] | ¥/$ __（+___%） | [什么信号确认] |
| **基准情景** | X% | [最可能路径] | ¥/$ __（+___%） | 维持现状 |
| **熊市情景** | X% | [核心悲观假设] | ¥/$ __（-___%） | [什么信号确认] |

> **L3 做三情景即可**（vs L4 的四情景含黑天鹅，vs L5 的概率加权目标价）。
> 三情景概率之和 = 100%。

#### 5.2 催化剂清单（精简版）

列出 **3-5 个关键催化剂**，每个一行：

| 催化剂 | 预计时间 | 方向 | 概率 |
|--------|---------|------|------|
| [催化剂 1] | Q?/月? | 利好/利空 | 高/中/低 |
| ... | | | |

#### 5.3 TSR 分解（必做）

```
预期总股东回报 (TSR) = 盈利增速 g + 股息率 d + 估值变动 Δ(PE)

g = ___%    （基准情景盈利增速）
d = ___%    （当前股息率）
ΔPE = ___%  （估值修复/压缩预期）
-----------------------------------------
TSR = ___%  

盈亏平衡增长率 g* = (1/PE) - d = ___%
→ 市场隐含预期增长率是 g* = ___%，合理吗？
```

#### 5.4 框架冲突仲裁（如有分歧）

如果不同框架结论矛盾，**必须**在此模块显式处理：

```markdown
### 框架冲突仲裁

| 框架 | 结论 | 方向 | 置信度 |
|------|------|------|--------|
| [框架A] | ... | 看多 | XX% |
| [框架B] | ... | 看空 | XX% |

**分歧解读**：[估值看便宜+成长看差 = 价值陷阱风险？还是...]
**仲裁结论**：根据标的属性，[价值/成长]维度权重更高 → 综合倾向 [方向]
```

> 所有框架一致 → 提醒"共识风险"（超额收益可能有限）。

#### 5.5 Kill-Switch 快速检查

L3 执行**简化版 Kill-Switch**（vs L4 的逐项深度审查）：

| Kill-Switch | 检查结果 | 是否触发 |
|-------------|---------|---------|
| 核心数据缺失/不可靠 | [有/无] | |
| 商业模式无法 30 秒说清 | [是/否] | |
| 管理层红旗 | [有/无，简述] | |
| 有息负债/EBITDA > 4x | [数值] | |
| 安全边际不足（当前价 > EPV） | [是/否] | |
| 收入 > 50% 来自单一来源 | [是/否] | |

> 触发任一"致命"级 → 立即降级为"观察"。触发"严重"或多个"警告" → 在报告中标注，降低置信度。

#### 5.6 范式归类

```markdown
### 范式归类

**正范式**：[质量复利 / 平台生态 / GARP / 低估催化 / 现金回报防御]（5选1）
**归类理由**：[一句话]

**负范式陷阱检查**：
- [ ] 倍数幻觉 — 正常化盈利 vs 报告盈利差异 > 30%？
- [ ] 杠杆驱动 — ROE 中财务杠杆贡献 > 50%？
- [ ] 伪成长 — 有机增长 < 总增长 50%？
```

## L3 效率指导原则

> **L3 的核心竞争力是"投入产出比"。以下原则确保不滑向 L4 的无底洞，也不退化为 L2 的粗糙。**

### 1. 框架精选而非穷举
4-6 个框架就是上限。如果多个框架结论高度重叠，**合并而非累加**。

### 2. 数据充分但不过度
完整三表 + 2-3 家对标公司。不做全行业横扫（那是 L4），不追溯 10 年历史（3 年足够）。

### 3. 深度适度
每个框架 300-500 字的分析。如果一个框架写了 1000 字还没写完 → 这个标的可能需要 L4。

### 4. 前瞻有框架
三情景分析必做，催化剂精简到 3-5 个，领先指标选最关键的 2-3 个。

### 5. 升级信号
在 L3 过程中，以下信号说明应升级至 L4/L5：
- **3+ 个框架结论严重冲突** → 升级 L4 全景验证
- **关键数据无法获取/矛盾** → 升级 L4 多源穿透
- **拟配置 >5% 组合权重** → 升级 L4 或直接 L5
- **用户追问"有没有其他角度"** → 考虑升级 L4

### 6. 降级信号
以下情况说明 L3 过度了，L2 足够：
- **用户只关心单一维度**（如"估值是高是低"）→ L2 即可
- **信息极度匮乏**（新上市/微盘股/无财务数据）→ L1-L2 概览
- **对话内快速追问**（不需要存档报告）→ L1-L2
