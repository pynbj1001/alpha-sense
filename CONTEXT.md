# 用户上下文档案（精简版）

> 用途：记录用户长期偏好与当前执行状态，供每次对话快速加载。  
> 原则：保留高价值、可执行信息；删除冗余流水账；持续迭代。

---

## 1) 用户画像（长期稳定）

- 身份：跨资产投资者，追求机构级研究质量。
- 核心市场：美股科技/成长。
- 覆盖市场：港股、外汇、中国利率债、宏观产业链。
- 风格标签：逆向 + 概率化 + 数据驱动 + 可执行。

---

## 2) 方法与输出偏好（高优先级）

### 2.1 研究方法

- 数据先行，框架随后：禁止凭记忆下财务结论。
- 关键数据要求双源交叉验证；若失败需明确披露，不可编造。
- 结论需带概率表达与反证条件（失效信号）。
- 重视二阶思考与可证伪性。

### 2.2 报告写作

- 偏好：专业投研表达，但避免模板化、机械化语言。
- 结构：前提 -> 证据 -> 推理 -> 结论（可追踪）。
- 风格：可读、叙事化、少空话；更像公众号长文——“先抛观点再上数据”，把关键机制讲明白。
- 关键段落需“数据+逻辑链”充分展开，不接受一句话带过。

### 2.3 输出格式

- 报告目录：`10-研究报告输出/`
- 命名格式：`[日期]-[类型]-[标的]-[框架].md`
- 交付习惯：生成 `.md` 后同步生成同路径 `.pdf`

---

## 3) 常用框架优先级（当前）

1. 格林沃尔德三重估值（资产 -> EPV -> 成长价值）
2. 贝叶斯拐点（概率更新）
3. 巴菲特护城河
4. 卡拉曼安全边际
5. 市赚率（PR）
6. 三重共振（周期×客户×产品）
7. 技术革命/指数引擎（成长赛道）
8. 动量框架（择时辅助）

---

## 4) 关键执行规则（当前有效）

- 债券策略先跑数据：`python bond_data.py --all`
- 债券报告必须包含：曲线、通胀、政策利率三项硬指标。
- 追踪任务以“新闻线索优先”，行情数据可选，不可阻塞产出。
- 重大偏差或数据异常必须显式标注（来源、时间、偏差说明）。
